# ClaimAI - AI-Powered Intelligent Claims Platform

**Modern, pastel-themed AI-driven claims processing platform** built to demonstrate enterprise-grade architecture for **Healthcare Insurance, Banking, and Ecommerce** domains.

![ClaimAI](https://apurupa810-dotcom.github.io/claim-ai-platform/)

## 🎯 Project Overview

ClaimAI is a full-stack demonstration project that showcases my real-world expertise as a **Senior Full Stack Developer**. 

It simulates an intelligent claims processing system with AI-powered analysis, fraud detection, and automated workflows — directly inspired by my experience at **Navy Federal Credit Union** (claims migration + MuleSoft APIs) and **American Express** (dispute processing).

**Live Demo:** [https://apurupa810-dotcom.github.io/claim-ai-platform/](https://apurupa810-dotcom.github.io/claim-ai-platform/)

## ✨ Key Features

- **AI Claim Analyzer** — Instant summarization and entity extraction using natural language processing
- **Real-time Fraud Detection** — ML-based risk scoring and anomaly detection
- **Auto-Processing Engine** — Intelligent routing and approval recommendations
- **Beautiful Pastel UI** — Modern, soft pastel design with excellent user experience
- **Enterprise Compliance Focus** — Built with PCI, PII, and SOC2 principles in mind

## 🛠️ Technology Stack

### Backend & Core
- Java 17 + Spring Boot
- Spring Cloud & Microservices
- MuleSoft-style API integration
- RabbitMQ / Kafka (event-driven)

### Frontend
- React + TypeScript
- Tailwind CSS (soft pastel theme)
- Responsive & accessible design

### Cloud & DevOps
- Azure Kubernetes Service (AKS)
- Docker & Helm
- Azure AI Services

### AI/ML
- Simulated OpenAI / LangChain integration for claim analysis
- Fraud detection model

## 🏗️ Architecture Highlights

- Microservices architecture using **Spring Boot**
- Event-driven design with messaging queues
- Secure API layer with OAuth2/OIDC patterns
- Containerized deployment ready for **Azure AKS**
- AI-powered decision engine for faster claim processing
